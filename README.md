# n8n Workflow Journey

This repository is a weekly collection of n8n workflows and automation projects.

Each week lives in its own folder so the files, workflow exports, and README files stay organized and do not clash with each other.

## Repository Structure

```text
.
|-- README.md
`-- week-02/
    |-- README.md
    `-- crypto price logger.json
```

## Weekly Projects

- [Week 02 - Crypto Price Logger](./week-02/README.md)

## How I Use This Repo Each Week

1. Create a new folder such as `week-03`, `week-04`, or `week-05`
2. Add that week's workflow files inside the folder
3. Add a `README.md` inside that same folder
4. Update this root README with a link to the new week
5. Commit and push the changes to the same repository

## Notes

- The root `README.md` is the main GitHub repo README.
- Each weekly folder should contain its own project-specific documentation.
- This keeps every week's work separate while still storing everything in one repository.
